# 架构约束 (Architecture Constraints)

* **客户端限制:** Echo 端必须使用 HarmonyOS Native (ArkTS) 开发，需遵循鸿蒙的后台任务管理规范。
* **技术栈约定:** 后端采用 Go 语言（搭配 Gin 框架）以提供稳定且轻量的 API 服务。
* **基础设施约束:** 考虑到成本控制，主要应用和数据库均部署在宿主机（个人电脑 / 软路由）上，并必须完全兼容通过第三方“内网穿透”隧道（如 FRP / Cloudflare Tunnel / Ngrok 等）以非固定公网 IP 的方式穿透暴露服务。这也意味着应用不能对依赖公网绝对高可用和超大带宽作出假设。
