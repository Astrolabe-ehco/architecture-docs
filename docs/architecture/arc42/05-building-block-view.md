## Level 2: 容器架构 (Container View)

将 Astrolabe 系统这个“黑盒”打开，我们可以看到其内部的系统边界。在这个层级，我们将鸿蒙手表作为独立的硬件采集触点，与手机 App 协同工作。

**核心技术栈：** HarmonyOS (ArkTS), Vue 3, Go (Gin), MySQL。

![](embed:Container)

### 容器职责说明：
* **Echo Watch (穿戴设备):** 负责无感采集用户的心率、步数、睡眠及系统的**运动锻炼数据**，并通过由软总线或 Health Kit 同步给手机。
* **Smart Scale (智能体脂秤/IoT):** 负责按天或按需求（约每两天一次）采集用户的体重与体脂等身体指标，同样通过 Health Kit 汇入手机主端。
* **Echo App (移动端):** 作为核心互动入口，提供番茄钟计时、任务标记功能记录，同时负责由于设备限制无法独立传网的**穿戴和体重设备的数据聚合**，向网关统一上报。
* **Main Backend (单体后端):** 集中处理业务逻辑，接收并结构化存储上述繁多来源的数据，同步调用远端 LLM 进行客观加主观的数据重组以生成专属的生活节奏指导报告。
