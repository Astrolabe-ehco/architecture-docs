# 质量需求 (Quality Requirements)

## 可靠性与可用性 (Reliability)
* **弱网容灾与断点续传:** 鸿蒙端在进入地铁无网环境、或者**宿舍后端物理机因断电、内网穿透隧道波动短时失联**时，时间记录与心跳数据必须暂存在设备本地的 SQLite 或首选项中，待网络或服务器恢复后，依据本地时间戳执行带有顺序的批量补偿上报。
* **服务容错:** 当外部 LLM API（如 OpenAI/通义千问）宕机时，AI 功能模块需能优雅降级，向用户返回缓存的提示或基于基本规则的标准建议，而不应导致整个后端的崩溃。

## 性能 (Performance)
* **API 延迟:** Go 主后端对于常规的数据上报与读取接口，不包含穿透隧道网络延迟的情况下 P95 响应时间必须小于 100ms。
* **批处理效率:** 每日的 AI 分析任务必须能在终端触发时平滑运行，并确保在几分钟内完成该日的个人报告生成与落盘。
